# Decisions — Лог архитектурных решений

Зачем этот файл: чтобы новый разработчик или новая сессия агента
не повторяла уже пройденные ошибки.

Формат записи:
```
## [Дата] — [Короткое название решения]
Решение: [что сделано]
Причина: [почему именно так]
Исключения: [если есть]
```

---

## Пример записей

### [Дата] — lucide-react вместо иконок из Figma
Решение: все иконки берём из lucide-react, не экспортируем из Figma.
Причина: Figma MCP не умеет читать SVG/PNG — API Error 400 и петля ошибок.
Исключение: логотип — inline SVG, так как он уникальный.

### [Дата] — picsum.photos для фото-плейсхолдеров
Решение: фото временно заменены на picsum.photos.
Причина: Figma MCP не читает изображения.
Когда заменить: перед финальной сдачей — экспортировать из Figma → src/assets/images/.

### [Дата] — i18next с самого старта
Решение: i18next настроен в самом начале, все строки сразу через t().
Причина: добавление i18n после написания компонентов требует переписывать все строки.
Правило: никогда не хардкодить строки, даже "временно".

### [Дата] — cn() для условных классов
Решение: все условные Tailwind классы через cn() (clsx + tailwind-merge).
Причина: без twMerge конфликтующие классы не переопределяются корректно.
